# CodexMeter 设计规格

## 目标

构建一个 macOS 13+ 原生状态栏应用，用极低的常驻资源占用展示：

- 本机 Codex 今日、本周、历史总 token 消耗量。
- Codex 五小时窗口和周窗口的剩余百分比、重置倒计时。
- 根据五小时窗口剩余百分比改变状态栏圆点颜色。
- 鼠标悬停或左键点击状态栏图标时展示用量浮层。

本机 token 统计只覆盖当前 Mac 上仍保留在 `~/.codex/sessions` 和 `~/.codex/archived_sessions` 中的 Codex 会话。额度窗口来自 Codex 写入本地会话事件的账户级数据。

## 不在范围内

- 不查询或汇总其他电脑上的 token 明细。
- 不调用未公开的 Codex 接口，不读取 `~/.codex/auth.json`，不持有账号凭据。
- 不提供主窗口、Dock 图标、账号登录、云同步或长期明细报表。
- 本阶段只生成本地可运行的 `.app`，不生成 DMG。

## 数据源方案

### 采用方案：本地 JSONL 增量索引

Codex 会话 JSONL 中的 `event_msg` / `token_count` 事件包含：

- `info.total_token_usage.total_tokens`
- `rate_limits.primary.used_percent`、`window_minutes`、`resets_at`
- `rate_limits.secondary.used_percent`、`window_minutes`、`resets_at`

选择该方案的原因：

- 不产生网络请求，不依赖登录态或额外权限。
- 不启动 Codex 子进程，避免 CLI 环境和版本变化带来的不稳定。
- 首次索引后只读取文件新增字节，常驻 I/O 很低。

未采用方案：

- 每次全量扫描：现有单个日志可达数十 MB，不满足低资源要求。
- Codex CLI 或内部接口：本机 CLI 当前受旧 Node 环境影响，且内部接口不是稳定契约。
- Codex 内部 SQLite：当前数据库没有稳定、公开的用量表结构。

## 统计口径

### Token 增量

每个会话中的 `total_tokens` 是累计计数。解析器按会话 ID 维护上一次累计值：

- 新值大于旧值：本次增量为两者之差。
- 新值等于旧值：这是重复状态事件，增量为 0。
- 新值小于旧值：视为计数器重置，本次增量为新值。
- 会话首次出现：本次增量为当前累计值。

增量按事件时间戳转换到系统当前时区后写入时间桶：

- 今日消耗：当天 24 个小时桶。
- 本周消耗：以周一为起点的 7 个天桶。
- 本机总量：所有历史桶求和；折线展示最近最多 12 个自然月的月桶。

统计值使用 `total_tokens`，即 Codex 事件中记录的输入与输出 token 总计口径。

### 额度窗口

从所有有效事件中选择时间戳最新的 `rate_limits`：

- `primary.window_minutes == 300` 对应五小时窗口。
- `secondary.window_minutes == 10080` 对应周窗口。
- 剩余百分比为 `max(0, min(100, 100 - used_percent))`。
- 重置倒计时由 `resets_at` 与当前时间的差值计算。

额度数据只在 Codex 写入新事件后更新。浮层必须展示“最后更新时间”；如果重置时间已过且没有更新事件，显示“等待 Codex 更新”，保留最后一次读数但标记为过期，不推断新的账户用量。

## 增量索引

索引保存到 `~/Library/Application Support/CodexMeter/usage-index.json`，只包含聚合数据和解析位置，不复制会话原文：

- 会话 ID、当前文件路径、文件标识、已读字节偏移。
- 上一次累计 token 值。
- 每个会话独立的小时、天、月贡献桶，以及由这些贡献合并出的展示汇总。
- 最新额度窗口及事件时间。

会话由 rollout 文件名中的会话 ID 标识。`sessions` 与 `archived_sessions` 出现同一 ID 时只处理一个副本，优先选择活动目录中的文件；文件移动到归档目录后沿用原偏移，不重复统计。

异常处理：

- 文件被截断或替换：先从汇总中撤销该会话已有贡献，再重新解析该会话。
- 最后一行尚未写完：保留未完成字节，下一次继续。
- JSON 损坏或字段缺失：跳过该行，不影响其他会话。
- 超长非 token 行：流式跳过，不将整行永久保留在内存。
- 索引损坏：删除索引并在后台重新建立。

## 更新策略与性能

- App 启动时先读取聚合索引，立即显示上次结果。
- 首次运行在 `utility` 优先级后台队列建立完整索引，状态栏显示灰色“索引中”状态，绝不阻塞主线程。
- 使用 vnode `DispatchSource` 监听会话目录与现有 JSONL 文件，收到变化后防抖约 1 秒；只有目录结构变化时才重建监听集合。
- 每 5 分钟执行一次轻量目录对账，处理漏掉的移动或新文件事件。
- 每 60 秒只更新倒计时文字，不扫描日志。
- 图表只在聚合数据变化时重算路径，不使用动画。
- 解析前先检查行内是否含 `token_count` 标记，只对候选短行执行 JSON 解码。

目标：空闲时无持续磁盘读取，除分钟倒计时外不做周期计算；常驻内存主要为状态栏 UI 和小型聚合索引。

## 状态栏图标

图标为 18 x 18 pt 非模板图像：实心状态圆点、低透明外环和一个微小高光点。颜色由五小时剩余百分比决定：

- `>= 70%`：绿色。
- `>= 30% 且 < 70%`：黄色。
- `>= 10% 且 < 30%`：橙色。
- `< 10%`：红色。
- 数据缺失、过期或正在建立索引：中性灰色。

## 浮层与交互

采用已确认的方案 D，浮层宽度约 320 pt：

1. 顶部显示“Codex 用量”和最后更新时间。
2. 第一行并排显示五小时与周剩余百分比、进度条、重置倒计时。
3. 下方纵向显示三张趋势卡：
   - 今日消耗：数值与小时折线，横坐标为 `00 时` 到 `现在`。
   - 本周消耗：数值与天折线，横坐标为 `周一` 到 `今天`。
   - 本机总量：历史总值与月折线，横坐标根据可用月份显示起点到 `本月`。
4. 指标区不显示“按小时 / 按天 / 按月”字样。

交互规则：

- 鼠标进入状态栏图标：显示非固定浮层。
- 鼠标离开图标与浮层后约 250 ms：关闭非固定浮层，避免移动过程中闪退。
- 左键点击：固定或取消固定浮层；固定后可在浮层内查看。
- 点击浮层外部：关闭固定浮层。
- 右键点击：显示“立即刷新”、“开机自启”和“退出 CodexMeter”。
- “开机自启”为勾选开关，默认关闭；点击后使用 `SMAppService.mainApp` 注册或取消注册，并按系统实际状态更新勾选状态。失败时显示错误提示，不伪造已启用状态。

## 架构

### CodexMeterCore

- `TokenEventParser`：把候选 JSONL 行解析为领域事件。
- `SessionUsageAccumulator`：计算累计计数的增量并处理重置。
- `UsageBuckets`：维护小时、天、月桶和展示快照。
- `RateLimitSnapshot`：计算剩余百分比、倒计时、过期状态与图标颜色。
- `UsageIndex`：可持久化的增量位置与聚合状态。

### CodexMeterCore 文件服务

- `UsageLogIndexer`：目录发现、去重、增量读取和索引持久化。

### CodexMeter App

- `UsageDirectoryMonitor`：低开销文件事件监听与 5 分钟对账。
- `UsageService`：后台刷新并向主线程发布不可变快照。
- `StatusItemController`：状态栏图标、悬停、点击和右键菜单。
- `UsagePopoverController`：AppKit 浮层布局。
- `SparklineView`：使用 `NSBezierPath` 绘制静态缩略折线。
- `LaunchAtLoginController`：封装 `SMAppService.mainApp` 状态查询、注册与取消注册。

UI 只依赖不可变 `UsageSnapshot`，不直接读取日志。

每个会话保留独立贡献桶是文件截断恢复和归档去重的必要条件；展示汇总始终由当前有效会话贡献合并得到。

## 错误状态

- 找不到 `~/.codex` 或没有会话：灰色图标，浮层显示“暂无本机 Codex 用量记录”。
- 没有额度事件：token 图表正常显示，额度卡显示“暂无额度数据”。
- 索引进行中：展示已完成部分与扫描状态，不冻结 UI。
- 单个文件读取失败：保留旧索引并记录诊断日志，下一次变化或对账时重试。

## 测试与验收

核心逻辑使用先测试后实现：

- 解析有效、缺字段、损坏和未完成 JSONL 行。
- 累计计数增长、重复、重置的增量计算。
- 会话移动、重复副本、文件截断后的去重和重建。
- 本地时区下小时、周一至周日、自然月分桶。
- 额度百分比边界：70、30、10、9.99 与未知/过期状态。
- 五小时分钟倒计时和周天数倒计时。
- 索引编码、解码和损坏恢复。
- 开机自启菜单状态映射和失败后状态回滚。

验收标准：

- App 启动后只出现在状态栏，不显示 Dock 图标或主窗口。
- 悬停和左键均可打开浮层，右键可刷新、切换开机自启或退出。
- 状态圆点严格按五小时剩余额度变色。
- 浮层内容、折线粒度和倒计时符合方案 D。
- 首次历史扫描和后续日志解析不阻塞主线程。
- 完整核心测试通过，release 构建和本地 `.app` 签名验证通过。
